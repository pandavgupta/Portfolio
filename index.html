Continuous Integration with Jenkins,GitHub and Docker

May 2020

In this project ,I have built a environment in git where we have two branch master and dev1 (developer1) ,when dev1 commit some job it will automatically pushed to Github repository . In jenkins we have build 3 jobs (job1,job2 and job3 ) when dev1 pushed file to Github then job1 will trigger automatically and download the dev1 repo and deploy it on a testing environment . After that role of Quality assurance team (QAT) will come in play who will check the proper functioning of features. When QAT approved then job3 will trigger which will merge the dev1 repo to master branch then job2 will trigger automatically which will fetch new file from Github and update it to older repo and deploy it on production environment.
